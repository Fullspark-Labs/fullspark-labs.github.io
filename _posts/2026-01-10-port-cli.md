Every developer knows this frustration: you try to start your dev server and get "EADDRINUSE: Port 3000 already in use."

We've built **port** to solve this problem.

## The Problem

Common scenarios:
- You forgot you had a server running
- Another process grabbed your port
- You need to quickly see what's using port 8080

## Installation

```bash
npm install -g @fullsparklabs/port
```

## Usage

### Kill a Process on Port

```bash
port kill 3000
```

Kills whatever process is using port 3000.

### List All Active Ports

```bash
port list
```

Shows all ports currently in use with process info.

### Reserve a Port for a Project

```bash
port reserve myapp 3000
```

Labels port 3000 as reserved for "myapp" - helpful when working on multiple projects.

### Check a Specific Port

```bash
port check 8080
```

Shows what's using port 8080.

## Why Use port?

- **Quick**: One command to kill stuck processes
- **Informative**: See all ports at a glance
- **Organized**: Label ports for projects

## Get Started

```bash
npm install -g @fullsparklabs/port
port kill 3000
```
