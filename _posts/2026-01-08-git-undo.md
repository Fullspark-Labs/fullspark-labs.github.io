# Introducing git-undo - Undo Anything in Git

We've all been there. You commit something, then realize it's wrong. Or you push to the wrong branch. Or you merge by accident.

`git-undo` lets you undo anything in git, safely.

## The Problem with Git

Git is powerful but dangerous. One wrong command and you can lose work:
- `git reset --hard` can delete commits
- `git push --force` can overwrite remote history
- `git merge` can introduce unwanted changes

## Installation

```bash
npm install -g @fullsparklabs/git-undo
```

## Usage

### Undo a Commit

```bash
git-undo commit
```

Shows recent commits and lets you choose which to undo. Provides confirmation before any destructive action.

### Undo a Push

```bash
git-undo push
```

Reverts pushed changes safely by creating a new commit.

### Undo a Merge

```bash
git-undo merge
```

Undoes a merge with confirmation.

### Drop a Stash

```bash
git-undo stash
```

Lists stashes and lets you drop selected ones.

### View History

```bash
git-undo history
```

Shows undo history so you can track changes.

## How It Works

- **Safe**: Always confirms before destructive actions
- **Interactive**: Visual selection of what to undo
- **Reversible**: Track undo history for reference

## Get Started

```bash
npm install -g @fullsparklabs/git-undo
git-undo commit
```
