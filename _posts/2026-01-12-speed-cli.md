Shell aliases are great... until you need to pass arguments. Traditional aliases can't do it.

We've built **speed** to fix that.

## The Problem

Regular aliases:
```bash
alias gpush="git add . && git commit -m"
# Can't pass the message as argument!
```

## Installation

```bash
npm install -g @fullsparklabs/speed-cli
```

## Usage

### Add a Shortcut

```bash
speed add ghpush -- git add . && git commit "$1" && git push
```

The `$1` gets replaced with your argument.

### Use It

```bash
speed ghpush "my commit message"
```

Expands to: `git add . && git commit -m "my commit message" && git push`

### List Shortcuts

```bash
speed list
```

### Remove a Shortcut

```bash
speed remove ghpush
```

## Multiple Arguments

```bash
speed add push_branch -- git push origin "$1" && git push origin "$2"

speed push_branch main dev
```

## Cross-Platform

Works on:
- macOS
- Linux
- Windows (PowerShell, cmd)

## Get Started

```bash
npm install -g @fullsparklabs/speed-cli
speed add gp -- git push origin main
speed gp
```
